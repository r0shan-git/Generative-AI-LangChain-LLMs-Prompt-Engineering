----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------
Problems With Current Short-Term Memory

• No role separation (no system / user / assistant distinction)
• Uses raw strings → weak conversation structure
• Memory grows indefinitely
• Can hit token limits
• API cost increases over time
• Response time becomes slower as history grows
• No trimming mechanism
• No summarization of old conversations
• Not production scalable
• No control over context window

Solution:
• Use structured messages (SystemMessage, HumanMessage, AIMessage)
• Implement memory trimming
• Use conversation summarization
• Store long-term memory in vector databases (FAISS, ChromaDB)
• Use ChatMessageHistory / RunnableWithMessageHistory / LangGraph Memory
